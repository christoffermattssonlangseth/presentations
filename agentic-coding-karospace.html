PART II

CONCEPT

What is Agentic Coding?

A paradigm shift from "AI suggests lines" to "AI executes tasks."

Traditional AI Coding

  • Autocomplete in your editor
  • Suggest the next few lines
  • You drive, AI assists
  • Context = current file

Agentic Coding

  • Full terminal + file system access
  • Reads, writes, and runs code
  • AI drives, you review
  • Context = entire project
TOOL

>_ Claude Code

Anthropic's CLI agent. Lives in your terminal, works across your entire project.

  • Runs in your terminal — reads files, writes code, executes commands
  • Full project context through file search, grep, and directory traversal
  • Iterative: runs tests, sees errors, fixes them autonomously
  • Extended thinking for complex architectural decisions
  • Git-aware: understands branches, diffs, commit history
TOOL

>_ Codex

OpenAI's CLI agent. Sandboxed execution with network-isolated containers.

  • Terminal-based agent with automatic sandboxing for safety
  • Reads your codebase, proposes and applies changes
  • Network-disabled by default — focused on local code tasks
  • Multi-file edits with unified diff review
  • Fast for focused, well-scoped implementation tasks

WORKFLOW

How I Used Both Tools

Not "one or the other" — each tool has its sweet spot.

Claude Code for...

Architecture decisions, complex features, debugging multi-file issues, refactoring, writing the 7K-line exporter

Codex for...

Focused feature additions, isolated component work, quick implementations where the scope was clear

*

Key insight: Use the right tool for the right job. Architect with Claude Code, implement scoped tasks with either.

PHASE 1

Architecture & Scaffolding

Starting from scratch: defining the project structure and data pipeline.

Defined the data model

h5ad in → SpatialDataset class → serialized JSON → embedded in HTML

Designed module boundaries

data_loader.py | exporter.py | cli.py | gui.py | annotations.py

Chose canvas over SVG/DOM

Canvas handles 100K+ cells without DOM overhead — critical for performance

Single-file output constraint

All JS, CSS, and data embedded inline — zero external dependencies

PART IV

Lessons
Learned

What worked, what didn't, and practical tips for your projects

WINS

What Worked Well

  • Rapid prototyping — Going from idea to working prototype in hours, not days
  • Fearless refactoring — "Restructure this 2000-line file into these modules" just works
  • Domain bridging — AI understood both Python/scanpy and browser JavaScript
  • Boilerplate elimination — Canvas rendering, event handling, UI state management
  • Learning accelerator — Used patterns and APIs I didn't know existed
CHALLENGES

What Didn't Work

  • Context limits — The 7K-line exporter sometimes exceeded context windows
  • Subtle regressions — Fixing one feature could silently break another
  • Over-engineering — AI sometimes adds unnecessary abstractions if unchecked
  • Domain-specific bugs — Spatial data edge cases required human domain knowledge
  • Testing gaps — Generated code often lacked comprehensive tests initially
TIPS

Prompting Strategies

How you talk to the agent matters as much as which agent you use.

Be Specific

"Add a split slider that composites two canvas layers with a draggable divider" beats "add compare mode"

Show Examples

Reference existing patterns in your code — "do it like the zoom handler in exporter.py line 3400"

Iterate Small

One feature per conversation. Don't ask for 5 things at once — you'll get 5 mediocre things

Review Everything

AI writes the first draft. You must understand it, test it, and own it. Never ship blindly.
